# CLAUDE.md

## Project Overview

Tuner is a statically served web application for tuning musical instruments. It uses the Web Audio API for microphone capture and tone generation, and the pitchy library for real-time pitch detection. No framework — vanilla JS with Vite.

## Commands

- `npm run dev` — Start dev server
- `npm run build` — Production build
- `npm test` — Run all tests (52 tests, Vitest)
- `npm run test:watch` — Tests in watch mode

## Architecture

**Audio pipeline**: microphone.js -> pitch-detector.js -> note-detector.js -> UI update
**Tone playback**: tone-generator.js (OscillatorNode with additive `PeriodicWave` + GainNode ADSR envelope -> shared DynamicsCompressorNode -> destination)
**State coordination**: Document-level CustomEvents (`play-tone`, `tone-start`, `tone-end`, `chromatic-mode-change`, `reference-pitch-change`)

All audio modules import from `src/audio/audio-context.js` which manages a singleton AudioContext. iOS Safari requires `initAudio()` to be called from a user gesture handler.

## Key Patterns

- **No framework**: Vanilla JS, ES modules, plain CSS. DOM manipulation via standard APIs.
- **Module singletons**: Audio modules (audio-context, microphone, pitch-detector, tone-generator) use module-scoped state. Not class instances.
- **CustomEvent communication**: Cross-module events dispatched on `document`. String selector emits `play-tone`; settings emits `reference-pitch-change`; chromatic toggle emits `chromatic-mode-change`. Tone generator emits `tone-start`/`tone-end` (with `frequency` in detail) so the string selector can reflect playback state on the speaker-icon buttons.
- **Configurable A4**: All music math functions in `src/utils/music.js` accept an optional `a4` parameter (default 440). Instrument frequencies in `src/data/instruments.js` store MIDI note numbers for recalculation.
- **Mount points**: UI components mount into predefined containers in index.html (`.gauge-container`, `.string-selector`, `.action-buttons`, `.app-header`). HTML structure lives in index.html, not generated by JS.

## Gotchas

- `OscillatorNode` cannot be reused after `stop()` — create a fresh instance per tone play.
- `exponentialRampToValueAtTime` cannot target 0 — use 0.0001 as the floor value.
- `AnalyserNode` must NOT connect to `ctx.destination` — only source -> analyser, otherwise mic feedback loop.
- `PitchDetector.forFloat32Array(fftSize)` must match the AnalyserNode's fftSize (4096).
- iOS Safari: AudioContext starts suspended, must call `resume()` after user gesture.
- getUserMedia requires HTTPS or localhost. For LAN dev access, `vite.config.js` sets `server.host: true` and uses `@vitejs/plugin-basic-ssl` for a self-signed cert so other devices on the network can get mic access.
- Ukulele (standard and D-tuning) uses re-entrant tuning — don't assume strings are ordered low-to-high. Low G ukulele tuning is linear. Baritone ukulele is its own top-level instrument (DGBE, linear).
- Half-step-down tuning uses sharp names internally (D#, G#) but flat names (Eb, Ab) for display.
- Trumpet and clarinet are transposing instruments; frequencies in `src/data/instruments.js` are stored as **concert pitches** (what the mic hears), not written pitches.
- Service worker lives in `public/` so Vite serves it at the root path. **Registration is gated to `import.meta.env.PROD`** in `main.js`; dev builds actively unregister any existing SW and clear its caches to prevent stale reloads. One-time cleanup: after pulling the SW gate, a hard refresh is needed to evict the previously-installed SW.
- Reference tone waveform uses a cached `PeriodicWave` selected by fundamental frequency (< 250 Hz → rich "low" partial stack; ≥ 250 Hz → lighter "high" stack). `PeriodicWave` auto-normalizes time-domain peak to 1.0. The goal of the partial stack is beat audibility (upper partials land in the ear's sensitivity peak), not timbre matching per instrument — aural piano tuning is the precedent.
- A single `DynamicsCompressorNode` on the tone-generator output is module-scoped and persistent; per-play `osc -> gain` subgraphs connect into it rather than to `ctx.destination` directly.
- SVG `transform-origin` must be set explicitly for CSS transitions on the gauge needle.
- Use `createElementNS` with SVG namespace for dynamically created SVG elements.

## Style

- Dark theme: background `#0f172a`, text `#e2e8f0`
- Colorblind-safe palette: teal/amber/orange (not red/green) + arrow icons as secondary indicators
- Mobile-first: 320px default, breakpoints at 768px and 1024px
- Touch targets: minimum 44px
- System font stack

## Testing

Tests cover `src/utils/music.js` and `src/audio/note-detector.js`. Audio modules that depend on Web Audio API are not unit-tested (would need browser environment mocking).
