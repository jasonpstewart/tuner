# Learnings: frontend

## Codebase Patterns
- Vite + vanilla JS, plain CSS, flexbox/grid. Mobile-first (320px+), controls in fixed footer.
- SVG gauge in tuner-gauge.js, string selector in string-selector.js, permission flow in permission-flow.js, settings in settings.js.
- Mount points in index.html: .gauge-container, .string-selector, .action-buttons, .app-header.

## Gotchas
- Colorblind-safe palette (blue/orange + lightness) — not red/green. Arrow icons as secondary indicator. (dispatch: task-13)
- Touch targets >= 44px for mobile. (dispatch: task-14)
- SVG transform-origin must be set in CSS, not SVG attribute, when using CSS transitions. Use style.transform = `rotate(Xdeg)`, not setAttribute('transform', `rotate(X cx cy)`). (dispatch: retro-bug-3)
- Use createElementNS with SVG namespace for dynamic SVG elements. (dispatch: task-13)

## Cross-Module Integration
- CustomEvent dispatch pattern: play-tone, stop-tone, tone-start, tone-end, reference-pitch-change on document. (dispatch: task-15, updated 2026-04-05)
- HTML structure lives in index.html — components mount into predefined containers, not generated by JS. (dispatch: task-14)
- Always verify UI features in the browser (npm run dev), not just build passes. CSS/SVG bugs are invisible to static checks. (dispatch: retro)

## State & Mode Patterns
- String-button state has TWO orthogonal indices: `selectedStringIndex` (sticky, set by user tap) and `detectedStringIndex` (transient, set by pitch detector's closest-string guess). NEVER conflate them. Detection never sets selection and never triggers tones — it only paints a visual hint ring. This preserves the user-intent override as an escape hatch when auto-lock is doing the wrong thing. (added: 2026-04-05, dispatch: retro-session)
- Merged-control tap semantics require an explicit state machine written out for every (current state × input) pair BEFORE coding. Ambiguous taps where "the same button means play-or-stop-or-select depending on invisible context" are the worst UX bug class. The tuner's string button has 4 states (idle, detected-only, selected-idle, selected-playing) and the table must cover all of them for tap-this and tap-different. (added: 2026-04-05, dispatch: retro-session)
- Before adding a mode toggle, ask if dual-readout can collapse the distinction. The Chromatic button was removed by showing target-string AND chromatic-note in the gauge simultaneously — the mode no longer matters because both pieces of info are always visible. (added: 2026-04-05, dispatch: retro-session)
- Sticky selection needs an explicit "no target" escape hatch. Users will select and need a way back to "any note" that doesn't require tapping another preset. The Any Note button is distinct-style (dashed border, italic) so it reads as a mode switch, not a tuning preset. (added: 2026-04-05, dispatch: retro-session)
- Any pulsing/animated UI state needs `@media (prefers-reduced-motion: reduce)` with a static alternative (solid ring instead of pulse). (added: 2026-04-05, dispatch: retro-session)
